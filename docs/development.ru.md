## Принципы работы

### Выбор окна затухания

Записи колоколов начинаются с короткого шумного атакующего перехода от удара языка. Анализатор пропускает первые `--attack-skip-ms` миллисекунд перед анализом. Оставшийся сигнал считается содержащим устойчивые затухающие обертоны.

Если запрошенный пропуск длиннее входного сигнала, утилита завершается с понятной ошибкой, а не молча выдаёт пустой спектр.

### Усреднение спектра

После пропуска атаки сигнал затухания разбивается на перекрывающиеся кадры с окном Ханна. Для каждого кадра вычисляется магнитудный спектр через `numpy.fft.rfft`, затем берётся среднее по всем кадрам. Это усреднение снижает случайный шум, сохраняя устойчивые спектральные особенности.

Размер FFT и шаг между кадрами задаются через `--fft-size` и `--hop-size`. Больший размер FFT даёт более точное частотное разрешение, но требует достаточной длины сигнала.

### Сглаживание

Усреднённый магнитудный спектр сглаживается фильтром Савицкого-Голая. Сглаживание уменьшает высокочастотные межбиновые флуктуации, не смещая положение пиков, что делает последующее обнаружение пиков более надёжным.

### Обнаружение пиков

Пики обнаруживаются с помощью `scipy.signal.find_peaks` в частотном диапазоне `[min_freq, max_freq]`. Два параметра управляют чувствительностью:

- **prominence** — насколько пик должен выделяться на фоне окружения.
- **distance** — минимальное число бинов между соседними пиками.

Для негармонических спектров колоколов `distance` должен быть достаточно малым, чтобы разрешить близко расположенные обертоны.

### Преобразование в ноты и центы

Каждая пиковая частота `f` независимо отображается на ближайший тон 12-тоновой равномерно темперированной системы:

```text
semitones = 12 * log2(f / 440)
note_index = round(semitones)
deviation_cents = 1200 * log2(f / (440 * 2^(note_index / 12)))
```

Имя ноты выводится из `note_index` с опорой на A4 = 440 Гц. Октава меняется на C.

### Негармонические обертоны колоколов

В отличие от струнных или деревянных духовых, обертоны колоколов не являются целыми кратными одной фундаментальной частоты. Поэтому каждый пик сопоставляется с 12-тоновой системой независимо, а не предполагается гармоникой. Именно поэтому анализатор сообщает имя ноты и отклонение в центах для каждого обнаруженного пика.

## Запуск тестов

Установите зависимости и запустите pytest:

```bash
python -m venv venv
venv\Scripts\pip install -r requirements.txt
venv\Scripts\python -m pytest tests/test_analyze_bell.py -v
```

Синтетические фикстуры генерируются автоматически во время выполнения тестов.

## Как добавить функциональность

1. Обновите `analyze_bell.py`, добавив новое поведение.
2. Если функция настраивается через конфиг, добавьте ключ в `CONFIG_SECTIONS` и `config_defaults()`.
3. Добавьте тесты в `tests/test_analyze_bell.py`.
4. При необходимости обновите пример конфига и `config/default.ini`.
5. Обновите англоязычную и русскоязычную документацию.
6. Перед коммитом запустите полный набор тестов.

## Архитектурные замечания

- **Извлечение первого канала:** Стереофайлы приводятся к каналу 0. Анализ многоканальных записей отложен.
- **Ленивый импорт matplotlib:** `matplotlib` и `pyplot` импортируются только при запросе визуализации, что ускоряет запуск для режимов с CSV-выводом.
- **Headless-бэкенд:** Когда задан `--no-show`, перед импортом `pyplot` вызывается `matplotlib.use("Agg")`, что позволяет работать без дисплея.
- **Приоритет конфигурации:** CLI > конфиг > встроенные значения. Путь `--config` извлекается в первом проходе, чтобы argparse мог установить значения по умолчанию из конфига.

## Устранение неполадок для разработчиков

**Установка `soundfile` на Windows**
- Убедитесь, что установлен рабочий C-компилятор, или используйте готовое колесо. Пакет использует libsndfile.

**Обработка 24-битного PCM**
- `soundfile.read(..., dtype="float64")` корректно читает 24-битные WAV-файлы и возвращает нормализованные значения в `[-1.0, 1.0]`.

**Проблемы с бэкендом matplotlib**
- Если в headless-окружении возникают ошибки бэкенда, убедитесь, что `--no-show` задан перед `--plot-save`.
- Бэкенд `Agg` принудительно устанавливается автоматически при наличии `--no-show`.
