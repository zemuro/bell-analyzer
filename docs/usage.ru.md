# Руководство по использованию

Это руководство описывает запуск анализатора обертонов колокольных сэмплов из командной строки.

## Запуск анализатора

Анализатор принимает один позиционный аргумент — путь к WAV-файлу.

```bash
python analyze_bell.py samples/bell.wav
```

Если входной файл не указан, по умолчанию используется `samples\bell.wav` относительно корня проекта.

## Полный справочник флагов CLI

| Аргумент | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `input` | путь | `samples\bell.wav` | Путь к входному WAV-файлу |
| `--output` | путь | `None` | Путь для CSV/таблицы; без него вывод в stdout |
| `--attack-skip-ms` | float | `100.0` | Длительность пропуска в начале (мс) |
| `--min-freq` | float | `50.0` | Минимальная частота пика (Гц) |
| `--max-freq` | float | `8000.0` | Максимальная частота пика (Гц) |
| `--prominence` | float | `0.005` | Минимальная выраженность пика |
| `--distance` | int | `20` | Минимальное число бинов между пиками |
| `--smoothing-window` | int | `11` | Длина окна сглаживания (нечётное >= 3) |
| `--fft-size` | int | `16384` | Размер FFT на кадр |
| `--hop-size` | int | `2048` | Шаг между кадрами |
| `--peak-count` | int | `None` | Максимальное число пиков (`None` = без ограничения) |
| `--format` | выбор | `csv` | Формат вывода: `csv` или `table` |
| `--visualize` | флаг | `False` | Открыть интерактивное окно matplotlib |
| `--spectrogram` | флаг | `False` | Псевдоним для `--visualize` |
| `--plot-save` | путь | `None` | Сохранить график в PNG |
| `--no-show` | флаг | `False` | Не открывать интерактивное окно |
| `--quiet` | флаг | `False` | Подавить текстовый CSV/табличный вывод |
| `--spec-nperseg` | int | `4096` | Длина окна STFT для спектрограммы |
| `--spec-noverlap` | int | `3072` | Перекрытие STFT |
| `--spec-nfft` | int | `4096` | Длина FFT для STFT |
| `--spectrum-floor` | float | `-50.0` | Минимум dB на графике спектра |
| `--spec-floor` | float | `-144.0` | Минимум dB цветовой шкалы спектрограммы |
| `--config`, `-c` | путь | `None` | Путь к пользовательскому INI-файлу |
| `--save-config` | путь | `None` | Записать эффективную конфигурацию и выйти |
| `--n-labels`, `-n` | int | `7` | Число подписанных пиков на графике спектра |

## Приоритет конфигурации

Значения разрешаются в таком порядке:

1. **Флаги CLI** — наивысший приоритет
2. **Конфигурационный файл** (`analyze_bell.ini` или файл через `--config`)
3. **Встроенные значения по умолчанию** — низший приоритет

## Примеры

### Базовый CSV-вывод

```bash
python analyze_bell.py samples/bell.wav
```

### Табличный вывод с пользовательным диапазоном частот

```bash
python analyze_bell.py samples/bell.wav --min-freq 200 --max-freq 4000 --format table
```

### Интерактивная визуализация

```bash
python analyze_bell.py samples/bell.wav --visualize --peak-count 12
```

### Headless-экспорт PNG

```bash
python analyze_bell.py samples/bell.wav --plot-save bell_report.png --no-show
```

### Загрузка пользовательского конфига

```bash
python analyze_bell.py samples/bell.wav --config my_bell.ini
```

### Сохранение конфига

```bash
python analyze_bell.py --save-config my_bell.ini
```

### Переопределение значения конфига из командной строки

```bash
python analyze_bell.py samples/bell.wav --config my_bell.ini --min-freq 200.0
```

## Советы по визуализации

- Используйте `--n-labels`, чтобы избежать перегруженных графиков. По умолчанию `7`; уменьшайте для плотных спектров.
- Используйте `--no-show` вместе с `--plot-save` на серверах и в CI.
- Подстройте `--spectrum-floor`, если тихие пики обрезаются или график выглядит пустым.
- Подстройте `--spec-floor`, если спектрограмма слишком шумная или слишком тёмная.

## Устранение неполадок

**Не найдено ни одного пика**
- Уменьшите `--prominence` (например, `0.001`).
- Уменьшите `--distance` (например, `10`) для близких обертонов.
- Расширьте диапазон `--min-freq` / `--max-freq`.

**Слишком длинный пропуск атаки**
- Уменьшите `--attack-skip-ms` или используйте более длинный сэмпл.

**Метки перекрываются**
- Уменьшите `--n-labels`.

**PNG пустой**
- Проверьте, что `--plot-save` получил корректный путь.
- В headless-окружении используйте `--no-show`.

**Конфигурационный файл не найден**
- Проверьте путь, переданный в `--config`.
- Используйте `--save-config`, чтобы сгенерировать корректный начальный файл.
